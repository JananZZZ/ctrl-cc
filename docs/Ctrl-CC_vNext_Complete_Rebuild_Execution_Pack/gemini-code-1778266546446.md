# Claude Code GUI 顶级集成方案规格说明书 (Tauri/Rust 版)

本规范旨在指导如何利用 Tauri 生态构建一个能够 110% 还原并超常增强 Claude Code CLI 的 GUI 应用。

## 1. 核心架构设计

### 1.1 后端引擎 (Rust + PTY)
* **依赖库**: `portable-pty` (跨平台伪终端)、`tokio` (异步处理)、`serde_json`。
* **核心策略**: 
    * 不能使用 `std::process::Command`，必须使用 `native_pty::PtySystem` 创建交互式会话。
    * **环境变量设置**: 必须注入 `TERM=xterm-256color` 和 `COLORTERM=truecolor`。
    * **非阻塞读取**: 使用字节级读取，切勿使用按行读取。

### 1.2 前端渲染与拦截 (TypeScript)
* **依赖库**: `xterm`, `xterm-addon-fit`, `strip-ansi`, `monaco-editor`。
* **双层架构**:
    1.  **底座层 (Base)**: `xterm.js` 实例（可见或隐藏），负责 100% 消化 ANSI 字符。
    2.  **智能层 (Intelligence)**: 挂载在 xterm.js 的屏幕缓冲区上的刮削器（Scraper），提取纯文本进行状态机判断。

## 2. 关键代码指导 (Rust 后端)
必须使用 `tokio` 异步读取，否则会锁死主线程，导致前端接收不到数据。

```rust
// 伪代码演示核心逻辑
let mut buf = [0u8; 1024]; // 关键：使用固定字节数组，不要 read_line
loop {
    match reader.read(&mut buf) {
        Ok(0) => break, // 进程结束
        Ok(n) => {
            let chunk = String::from_utf8_lossy(&buf[..n]).to_string();
            window.emit("pty-data", chunk).unwrap();
        }
        Err(_) => break,
    }
}
